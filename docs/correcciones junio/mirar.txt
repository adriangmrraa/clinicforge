## 2. Mensajes sin respuesta + Forzar Procesamiento IA no funciona (FIXED)

**Causa raíz:** NameError por `import re` condicional dentro de `process_buffer_task`.

En `orchestrator_service/services/buffer_task.py`, línea 3666:
```python
if "[LOCAL_IMAGE:" in response_text:
    import re  # ← REDUNDANTE Y DAÑINO
```

`re` ya está importado a nivel módulo (línea 9). Pero este `import re` condicional hace que Python trate `re` como **variable LOCAL** de toda la función `process_buffer_task`. Como está dentro de un `if`, cuando la condición es false o no se ejecutó todavía, `re` como local nunca se asigna.

Después, el list comprehension en línea 1373:
```python
_guardian_phones = [re.sub(r'\D', '', p) for p in [external_user_id]]
```
corre en un scope anidado, busca `re` en el scope contenedor (`process_buffer_task`), y como `re` es local sin asignar, explota con:
```
NameError("cannot access free variable 're' where it is not associated with a value in enclosing scope")
```

**Síntomas que causaba:**
- Mensajes de pacientes que quedan sin respuesta aunque el agente esté activo
- Botón "Forzar Procesamiento de IA" que no produce respuesta (misma code path: admin_routes.py línea 2326 → process_buffer_task)

**Fix:** Eliminado el `import re` redundante de línea 3666.
