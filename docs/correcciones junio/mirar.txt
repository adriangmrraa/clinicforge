# Cosas para mirar

## 1. ✅ Recordatorio preview: profesional hardcodeado (FIXED)

En `orchestrator_service/routes/playbook_routes.py`, línea 270:
- `"professional": "Laura Delgado"` estaba hardcodeado, siempre mostraba "Con Laura Delgado" en la preview del reminder sin importar qué profesional tenía el turno.
- Fix: se agregó `LEFT JOIN professionals pr ON a.professional_id = pr.id` a la query, se traen `pr.first_name/professional_last_name`, y se usa el nombre real del profesional.

## 2. Mensajes sin respuesta intermitentes (FIXED)

**Causa raíz:** `import re` condicional dentro de `process_buffer_task` (línea 3666) shadoweaba el import global de `re`, rompiendo los list comprehensions cuando el `if` era false.

**Síntomas:** 2-3 chats no respondidos, Forzar Procesamiento sin efecto.

## 3. Mensajes sin respuesta TOTAL — post-deploy (FIXED)

**Causa raíz:** El bloque multi-agent dispatch (pre-existente en el working tree) hacía que el solo retry loop hiciera `break` inmediato porque `response_text` ya tenía valor. Pero `response` solo se definía DENTRO del loop. Post-loop, `response.get("intermediate_steps", [])` explotaba con `UnboundLocalError`.

**Por qué pasó AHORA:** Antes del fix #2, `process_buffer_task` explotaba en el `re` NameError ANTES de llegar al multi-agent dispatch. Con el fix #2, el código llega al multi-agent, ejecuta OK, pero se topa con este segundo bug que es taba latente.

**Fix:** Inicializar `response = {}` antes del loop. Cuando multi-agent responde, `response` queda como `{}` y los `.get()` post-loop devuelven `[]` sin error.

```
process_buffer_task_fatal_error | UnboundLocalError("cannot access local variable 'response' where it is not associated with a value")
```

## 4. ✅ Error email de confirmación de pago (FIXED, en este commit)

**Error:** "column t.phone does not exist"

**Causa:** `email_service.py` referenciaba `phone` como columna directa de `tenants`, pero no existe. El teléfono de la clínica se guarda en `config->>'phone'` (JSONB).

**Fix:** Cambiado `t.phone` a `t.config->>'phone' as phone` en las dos queries (welcome email + payment confirmation).
