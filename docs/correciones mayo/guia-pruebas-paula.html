Flujo de referencia

Flujo ideal completo — Implantes con OSDE

PACIENTE
Quiero implantes
PAULA
Claro. Lo ideal es realizar primero una evaluacion para ver que opcion es la mas adecuada para vos.

Tengo estos turnos disponibles:
1 — Martes 12/05 — 10 hs
2 — Jueves 14/05 — 10 hs

Cual te queda mejor?
PACIENTE
Martes. Tengo OSDE
PAULA
Perfecto. Te agendo entonces para el martes 12/05 a las 10 hs.

Si, trabajamos con OSDE. La consulta tiene coseguro y luego vemos especificamente la cobertura segun el tratamiento.

Te envio la direccion y los datos para confirmar el turno.
Puntos clave del flujo
1. Ofrece evaluacion + turnos inmediato (sin repetir "te ayudo a coordinar")
2. "Martes" con un solo martes disponible = seleccion inequivoca
3. Confirma turno + responde sobre OSDE en un solo mensaje
4. Usa "la Dra." + apellido (nunca nombre solo)
5. Continua con direccion, sena, anamnesis, sin re-ofrecer
Lo que NO debe pasar
/ "Con solo Martes no queda claro cual preferis"
/ Derivar a humano por combinar turno + OS
/ Ignorar la pregunta de OSDE
/ Volver a ofrecer turnos despues de confirmar

Metodologia

Tips para testear

1
Conversaciones separadas
Cada escenario debe testearse en una conversacion NUEVA. Si se mezclan, el contexto de un test puede afectar otro.
2
Probar en secuencia
Ademas de tests individuales, probar flujos encadenados: Escenario 2 (turno) / 3 (datos) / 8 (slot+OS). Esto verifica que el flujo completo funciona.
3
Anotar textualmente
Copiar y pegar exactamente lo que Paula responde. No parafrasear. El texto exacto permite comparar contra lo esperado y detectar variaciones.
4
Capturas si algo falla
Si un escenario no responde como se espera, tomar screenshot del chat completo. La captura exacta es la mejor forma de diagnosticar y corregir.
